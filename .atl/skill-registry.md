## Compact Rules (auto-resolved)

### NestJS Best Practices

- Use **constructor injection** for dependencies (no property injection)
- Follow **import order**: NestJS core → External → Shared (`@common/*`) → Local DTOs
- **Files**: `kebab-case.ts` | **Classes**: `PascalCase` | **DTOs**: `PascalCase` + `Dto` suffix
- Modules use `@Module({ imports: [...], controllers: [...], providers: [...], exports: [...] })`
- Repository pattern for data access
- Use Mongoose transactions via `TransactionService.withTransaction()`
- Guards for auth (`JwtAuthGuard`), roles (`RolesGuard`)
- Decorators: `@Public()`, `@Roles(...roles)`

### Code Quality

- **Lint**: `npm run lint` — ESLint with auto-fix
- **Format**: `npm run format` — Prettier
- **Test**: `npm run test` — Jest with `ts-jest`
- **Build**: `npm run build` — NestJS compiler

### SDD Workflow

- `proposal` → `specs` → `tasks` → `apply` → `verify` → `archive`
- Use Given/When/Then format for scenarios
- RFC 2119 keywords (MUST, SHALL, SHOULD, MAY) in specs
- Group tasks by phase with hierarchical numbering (1.1, 1.2, etc.)
